Extended Lovely String
Time Limit: 1 second

Problem

The values of uppercase alphabets A, B, C, ..., Z are assigned to be 1, 2, 3, ..., 26 respectively. The values of lower alphabets a, b, c, ..., z are assigned to be 27, 28, 29, ..., 52 respectively.

An extended lovely string (XLS) is a string that meets all of the following requirements:

  • It consists entirely of uppercase and/or lowercase alphabets.
  • It consists of at least one character.
  • The inequality
    v1 x 9 <= v2 x 8
    holds for any two adjacent characters, where v1 is the value of the former character and v2 is the value of the latter character.

For example, De is an XLS, but EF is not an XLS.

Candy has a lexicon. Her lexicon is the list of all XLS in ascending ASCII order along with their ordinal numbers starting from 1. For example, in the lexicon A is the extended lovely string number 1 and AB is the extended lovely string number 2.

Note: if an XLS is a prefix of another XLS, then the former one appears before the latter one in the lexicon. For example, ABE appears before ABEFG.

However, since she is too lovely, Candy does not know how to look up an XLS in the lexicon. She asks you to write a program that determines the ordinal number of an arbitrary given lovely string. There are no more than 231 XLS in the lexicon.

Input

The input consists of only one line which contains a valid extended lovely string.

Output

The output consists of only one line which contains the ordinal number of that extended lovely string.

Sample Input

ABCDEFGH

Sample Output 

8
